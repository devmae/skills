# 하네스 감사 — 주기적으로 (그리고 모델 업그레이드 후마다) 실행

각 하네스 구성요소(스프린트 계약, evaluator, 작업 템플릿 등)에 대해:
1. 그 구성요소를 끈다.
2. 전체 features.json 검증을 실행한다.
3. 출력 품질이 떨어졌는가?
   - 예  → 유지한다.
   - 아니오 → 삭제한다. 매 실행마다 토큰만 소모하는 죽은 무게다.

시간에 따른 decay 추적:
| 구성요소               | 여전히 핵심인가? | 마지막 테스트 | 조치 |
|------------------------|------------------|---------------|------|
| 스프린트 분해          |                  |               |      |
| 스프린트별 evaluator   |                  |               |      |
| 작업 템플릿            |                  |               |      |

목표: features.json을 여전히 통과시키는 가장 단순한 하네스.
